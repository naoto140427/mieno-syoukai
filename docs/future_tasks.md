## 🚀 最優先タスク (High Priority)

### 1. 認証フロー（マジックリンク）の改善とローカル開発環境の制限回避
ローカル環境でのログイン検証時に、Supabaseの無料枠メール送信制限（`email rate limit exceeded`）が発生したため、およびマジックリンクの別タブ遷移問題への対策が必要です。

*   **課題点**:
    1. 送信されたマジックリンクをクリックすると別タブでリダイレクトされ、元のタブが未ログインのままになってしまう。
    2. Supabaseの無料枠では短時間に何度もメール送信をテストできず、レートリミットに達してしまう。
*   **具体的な解決アプローチ**:
    *   **アプローチA（開発用のバイパス手段）**: 
        *   ローカル開発環境（`process.env.NODE_ENV === 'development'`）に限り、メール送信をバイパスして、特定の固定パスワード（またはデバッグ用OTP）で即座にセッションを確立できるバックドアや「デバッグログインボタン」を一時的に用意する。
    *   **アプローチB（Supabase Local Emulator の利用）**:
        *   Supabase CLIを使用してローカルエミュレータを立ち上げ、ローカル環境内でメール送信やインボックス確認（Inbucket）を完結させる。これにより制限を気にせずテスト可能。
    *   **アプローチC（`onAuthStateChange` の導入）**:
        *   元のタブでもログインを自動検知してリダイレクトさせるため、ログイン画面に `supabase.auth.onAuthStateChange` のリスナーを仕込み、別タブでの認証完了時に自動でダッシュボードへ遷移させる。

## 💡 コンテンツ再設計・機能検討タスク (Content & Strategy)

サイト全体の価値を高めるため、情報量が不足しているページや、用途の整理が必要なページのコンテンツ・機能を再定義します。

### 8. 「事業領域 (`/services`)」と「沿革 (`/history`)」のコンテンツ拡充
*   **現状の課題**: 情報が少なく、単純に内容が薄く見えてしまう。
*   **再設計案**:
    *   **事業領域**: 三重野商会が提供するモビリティ価値（戦略的展開、広域ロジスティクス）を、アイコンや美しいイラストを交えたストーリー形式のカードレイアウトで説得力を持たせる。
    *   **沿革**: タイムラインUIを極限まで美しく（スクロールアニメーションで歴史が浮かび上がる等）実装し、過去の主要作戦（ツーリング、機体導入）を魅力的なドキュメントのように演出して「薄さ」を「洗練さ」へ転換する。

### 9. 「広域兵站 (`/logistics`)」の全国開拓状況・走破ルートデータベース化
*   **決定方針**: 削除案は廃止し、**「全国開拓状況」と「走破ルートデータベース」を融合させたビジュアル特化ページ**としてリニューアルします。
*   **実装内容・追加アイデア**:
    *   **トータル走行データの可視化**:
        *   「作戦記録（Archives）」または「広域兵站（Logistics）」に、これまでの累積走行実績（例: トータル走行距離 `24,530 km`、踏破都道府県数 `12 / 47` など）を、洗練された巨大タイポグラフィと数値カウンターアニメーションでダイナミックに表示する。
    *   **走破ルートデータベース & マップ**:
        *   過去の主要作戦（ツーリング）で走行した具体的なルート（例: 大分・九重連山、山梨・富士五湖など）を、マップや走行データ（難所、給油スポット、標高データ）と共に美しくデータベース化。
        *   「この道はすでに走破済みである（作戦完了）」というステータスが直感的に伝わる、ゲーミフィケーション（開拓・踏破）要素を取り入れたプレミアムなUI/UXを構築する。

## 🛠️ 開発お役立ちコマンド (Resend CLI の使い方)

ローカル環境にて **Resend CLI** のインストールとAPIキーの連携（macOS Keychainへの安全な保存）が完了しています。開発やテストの際に以下のコマンドが役立ちます。

### 1. 接続状態の確認
CLIが正常に認証されているか、現在のステータスを確認します。
```bash
resend whoami
```

### 2. コマンドラインからのテストメール送信
お問い合わせの送受信テストなどを、プログラムを起動せずにターミナルから直接実行できます。
```bash
resend emails send \
  --from "info@mieno-shokai.com" \
  --to "your-email@example.com" \
  --subject "CLIからのテストメール" \
  --text "Resend CLIの連携が正常に完了しました！"
```

### 3. ドメイン情報の確認
現在登録されている送信ドメインの検証ステータスなどを一覧表示します。
```bash
resend domains list
```

### 4. APIリクエストログの確認
最近送信されたメールの配信ステータスやAPIエラーの履歴を確認します。
```bash
resend logs
```
