## 🚀 未完了タスクロードマップ ＆ 詳細仕様

### 🛠️ 開発環境 ＆ 基盤改善

#### 1. 開発環境用の認証改善（マジックリンクのバイパス ＆ ログイン自動同期）
*   **課題**: Supabaseの無料枠メール送信制限（`email rate limit exceeded`）により、ローカル環境でのログインテストが頻繁にストップする。また、マジックリンクをクリックすると別タブで認証が開くため、元のタブが未ログインのまま取り残される。
*   **UI/UXイメージ**: 
    *   開発環境限定で、ログイン画面下部に「【開発者用】パスワード認証でログイン」というトグルスイッチが表示される。
    *   別タブでログインが完了した際、元のタブも自動的に「ログイン成功」を検知し、アニメーションしながらダッシュボードへリダイレクトする。
*   **技術アプローチ**:
    *   `process.env.NODE_ENV === 'development'` の場合のみ、マジックリンク送信をスキップして `supabase.auth.signInWithPassword` を直接実行するトグルを実装。
    *   `supabase.auth.onAuthStateChange` のリスナーを `AdminLoginClient.tsx` に実装し、セッション確立時に自動でリダイレクトを実行。

### 🎨 UI/UX ＆ フロントエンド改善（Cupertino Minimal への統一）

#### 2. 「在庫情報 (`/inventory`)」の UI/UX 改善
*   **UI/UXイメージ**:
    *   Cupertino Minimal に基づく白背景（`#F5F5F7`）と、大きく余白をとった美しい Bento Box カードレイアウト。
    *   在庫の各行には車両のミニサムネイルを表示し、ステータス（稼働中、整備中など）は主張しすぎない洗練されたグレーやブルーのドットバッジで表現。
    *   操作時の `confirm()` / `alert()` を完全に排除。行をホバーした際に「編集」「削除」アクションがスッと浮き上がり、削除を選択するとその行が薄い赤にフェードしながらインラインで「本当に削除しますか？ [はい] [キャンセル]」と切り替わるスムーズなインライン確認UIを実装。
*   **技術アプローチ**:
    *   状態管理（`isDeletingId` などの State）を利用したインラインの切り替え。
    *   操作の実行中（Pending）はボタンを非活性化し、ローディングスピナーをアニメーション表示。

#### 3. 「広域兵站 (`/logistics`)」の全国開拓状況＆ルートデータベース化
*   **UI/UXイメージ**:
    *   **巨大オドメーター**: ページ上部に、パタパタ時計（スプリットフラップ）やドラムメーターを模した走行距離カウンターを設置。初期マウント時に `0 km` から `24,530 km` などへドラムが縦スクロールしながらカウントアップ。
    *   **開拓日本地図**: シンプルなSVG日本地図を描画。走破済みの県がブルーグレーで塗られ、ホバーするとピンが立って「走破回数」「代表ルート名」がふんわりポップアップする。
    *   **走破ルートカード**: 過去のツーリング作戦をカード化。カード内にはそのルートの標高アップダウンを表現した美しい1本のSVGライン（標高プロファイル）を描画し、スクロールに合わせてラインがサーッと描画される。
*   **技術アプローチ**:
    *   `framer-motion` の `useMotionValue` と `animate` でドラムロールを制御。
    *   SVGの `strokeDasharray` と `strokeDashoffset` によるパス描画アニメーション。

#### 4. 「事業領域 (`/services`)」＆「沿革 (`/history`)」＆「IR情報 (`/ir`)」のプレミアム・コンテンツ拡充
*   **UI/UXイメージ**:
    *   **沿革**: 縦方向の極細ラインがスクロール量に応じて伸びていき、それぞれの歴史ポイント（西暦や作戦名）が左右から交互に滑らかにフェード＆スライドインするタイムラインUI。
    *   **IR情報**: AppleのESGレポート（環境報告書）のような贅沢なホワイトスペースと巨大で美しい数字（タイポグラフィ）。累積走行距離や年間ツーリング実績の推移を、ライブラリを使用しない極限まで軽量なSVG円グラフ・棒グラフでアニメーション描画。
*   **技術アプローチ**:
    *   `framer-motion` の `useScroll` を利用したスクロール連動のアニメーション制御。
    *   ピュアCSS ＆ SVGによる高解像度かつ軽量なインフォグラフィックスの実装。

### 📡 システム拡張 ＆ 高度な機能

#### 5. 各機体（車両）の「個別オドメーター（走行メーター）」の連動
*   **UI/UXイメージ**:
    *   `/units` の一覧および各詳細ページに、機体ごとの累積走破距離を示すデジタルフリックメーターを埋め込む。
    *   管理画面（Admin）から各車両のオドメーターの数値を手動または走行ログ追加時に自動で更新できる仕組み。
*   **技術アプローチ**:
    *   Supabaseの `units` テーブルに `odometer` (INTEGER, デフォルト 0) を追加。
    *   走行ログ（`logs`）が追加された際、その走行距離を元の `units.odometer` に加算して自動同期するDBトリガー、または Server Action 側のトランザクション処理の実装。
    *   データ更新時は `revalidateTag` でキャッシュをパージし、CDN（Edgeキャッシュ）側も瞬時に最新のメーター値へ切り替える。

#### 6. 管理コンソール（Admin）を「作戦指揮統制室 (Command Center)」に変革する
*   **UI/UXイメージ**:
    *   管理者画面（`/admin`）を、上記「Apple Pro Apps & visionOS ダークグラストーン」の基準に沿ってフルリニューアル。
    *   ダッシュボード上部には「システム稼働ステータス」「エージェント接続数」などがHUDモニター風に整列。
    *   中央には「直近の通信監視（未返信のお問い合わせ）」「作戦行動ログ（最近の更新フィード）」が自動スクロールするログストリームとして配置。
    *   クイック入力エリア（コマンドバー）から、マウスを使わずショートカットキーで瞬時に各種データの更新画面を呼び出し可能にする。
*   **技術アプローチ**:
    *   Tailwindのダーク透過カラーと `backdrop-blur-xl` による重層的なグラスデザイン。
    *   アクティビティフィードの非同期更新とスムーズな追加アニメーション。

#### 7. 「View Transitions API」を活用した極上のページ間トランジション
*   **UI/UXイメージ**:
    *   車両一覧 (`/units`) のカードをクリックすると、カード内の車両画像が滑らかに拡大・移動しながら詳細ページ (`/units/[slug]`) の巨大なヒーローヘッダー画像へと直接変形して繋がるシームレスなトランジション。
*   **技術アプローチ**:
    *   Next.jsの画面遷移において、ブラウザ標準の View Transitions API（または `framer-motion` の `layoutId` による共有レイアウトアニメーション）を組み込み、ページをまたぐシームレスな体験を実現。

#### 8. [超特化型] Racing Spec Native iOS Logger App (別スタック共同開発)
*   **UI/UXイメージ**:
    *   バイク/クルマのツーリング・サーキット走行を記録する専用ネイティブアプリ (Swift/SwiftUIベース)。
    *   `ROADSTOCK` のような「GPSログ、自動停止検知、イベント記録」と、`LAP & LEAN` のような「高周波演算による精密なバンク角/Gフォース検知」を統合した完全上位互換版。
    *   UIは「Tactical Apple（極限まで削ぎ落とされた美しい機能美）」を踏襲。
*   **技術アプローチ**:
    *   **CoreLocation & CoreMotion**: Expo/React Native等のクロスプラットフォームのパフォーマンス限界を突破するため、Apple純正ツールを活用しバックグラウンドでの安定稼働を実現。
    *   **アーキテクチャ**: `mieno-syoukai-logger`（仮）など別ディレクトリ/別リポジトリにて開発し、VercelのWebビルドに影響を与えないように完全に分離。
    *   **DB共有**: Webアプリ側のデータベース（Supabase）をiOSからも直接共有。走行完了時にJSONをSupabaseまたはNext.jsの専用APIエンドポイントへ高速同期し、ダッシュボードへデータを流し込むエコシステムを構築。
